# Quality Checklist

- [ ] All files in scope were read and analyzed
- [ ] Findings classified by severity (critical/high/medium/low/info)
- [ ] No duplicate or redundant findings
- [ ] Each finding includes file path, line number, and description
- [ ] Remediation suggestions are actionable and tested
- [ ] False positives are minimized and noted
- [ ] Security findings are prioritized over style issues
- [ ] Report is structured and readable
- [ ] Test coverage gaps are identified
- [ ] Performance antipatterns are flagged
- [ ] Hardcoded secrets are reported immediately
