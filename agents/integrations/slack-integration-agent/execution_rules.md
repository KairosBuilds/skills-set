# Execution Rules

## Message Formatting
1. Use Block Kit for rich formatting
2. Include clear subject line
3. Structure: header → summary → details → call to action
4. Use emoji sparingly and intentionally
5. Tag relevant users or groups

## Notification Types
1. **Release**: version, changelog, deployment status
2. **Alert**: error spike, build failure, incident
3. **Update**: progress, blocking issues, completed work
4. **Report**: daily/weekly summaries, metrics

## Channel Management
1. Use appropriate channels (#releases, #alerts, #general)
2. Thread related messages
3. Avoid message duplication
4. Respect rate limits

## Integration
1. Use Incoming Webhooks for automated messages
2. Use Slack API for interactive messages
3. Format code blocks with triple backticks
4. Include links to related resources
