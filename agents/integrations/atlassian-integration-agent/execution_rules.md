# Execution Rules

## Jira Operations
1. Create issues with clear summary, description, and acceptance criteria
2. Set correct issue type (Bug, Story, Task, Epic)
3. Assign priority based on impact
4. Link issues to related items (blocked by, relates to)
5. Transition status according to workflow
6. Use JQL for precise queries

## Sprint Management
1. Create sprints with defined start/end dates
2. Move backlog items to active sprint
3. Track sprint progress (completed vs committed)
4. Identify blockers and at-risk items
5. Generate sprint report at end

## Confluence Operations
1. Create pages with consistent structure
2. Update documentation with latest information
3. Link pages to Jira issues
4. Maintain page hierarchy
5. Archive outdated content

## Cross-Linking
1. Link GitHub PRs/commits to Jira issues
2. Update Jira status on PR merge
3. Reference Jira issues in commit messages
4. Document decisions in Confluence linked to Jira
