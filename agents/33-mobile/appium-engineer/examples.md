# Examples — Appium Engineer

## Example 1: Login Flow E2E
```
Task: Automate login, OTP verification, and post-login state
Output: Page objects for login screen, OTP screen, home screen; test with valid/invalid credentials
```

## Example 2: E-Commerce Checkout
```
Task: Test complete purchase flow — browse, add to cart, checkout
Output: Product list page, cart page, checkout page objects; parameterized test for multiple products
```

## Example 3: Gesture Automation
```
Task: Automate swipe, pinch, zoom, and long-press gestures
Output: Gesture helper class using TouchAction/W3C Actions; tests for image gallery
```

## Example 4: Hybrid App Testing
```
Task: Test WebView content inside native app
Output: Context switching (NATIVE_APP to WEBVIEW), element interaction inside WebView
```

## Example 5: Deep Link Testing
```
Task: Verify deep link routing opens correct screens
Output: Test launching app via URL scheme, verifying navigation state
```

## Example 6: Push Notification Validation
```
Task: Validate push notification delivery and tap behavior
Output: Notification permission handling, notification tap routing verification
```

## Example 7: Parallel Execution
```
Task: Run tests in parallel on multiple Android devices
Output: Pytest-xdist configuration, device-specific capabilities, parallel test reports
```

## Example 8: Visual Regression
```
Task: Compare screenshots for UI regression detection
Output: Appium screenshot capture, pixel comparison with OpenCV, diff overlay generation
```
