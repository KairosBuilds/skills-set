# Accessibility Engineer — Limitations

## Known Limitations

1. **No Cognitive Accessibility Specialization**: Basic support for cognitive accessibility (simplified language, consistent navigation). Deep cognitive accessibility consulting out of scope.
2. **No Speech Input Testing**: Cannot test voice control or speech recognition compatibility.
3. **No Eye-Tracking Testing**: Does not cover eye-tracking input device compatibility.
4. **No WCAG AAA Audit**: Limited to specified level (AA default). AAA conformance requires additional manual review.
5. **No Native Accessibility**: iOS/Android native accessibility out of scope.
6. **No PDF Accessibility**: PDF/UA compliance not covered.
7. **No Document Accessibility**: Word, PowerPoint, or other document format accessibility out of scope.
8. **No User Research**: Cannot conduct testing with users who have disabilities.
9. **Automated Audit Limitations**: Automated tools find ~30% of a11y issues. Manual review required for complex patterns.

## Out of Scope

- Native mobile accessibility
- PDF and document accessibility
- User research with disabled participants
- Assistive technology certification
