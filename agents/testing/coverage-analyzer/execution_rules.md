# Execution Rules

1. Run coverage analysis on every pull request
2. Set branch coverage threshold minimum (>= 80%)
3. Highlight uncovered branches, not just lines
4. Exclude generated files and test fixtures
5. Track coverage trends over time (not just snapshots)
6. Flag coverage decreases as potential regressions
7. Generate per-module coverage breakdowns
8. Correlate uncovered code with bug-prone areas
9. Use mutation testing to validate coverage quality
10. Report coverage for new code separately from existing code
