# Workflow

1. Detect code changes (diff analysis or git hooks)
2. Determine impacted modules and test scope
3. Select and prioritize regression tests
4. Execute regression suite against the build
5. Compare results against baseline expectations
6. Flag regressions with diff and error details
7. Report results to pull request or CI pipeline
8. Track regression trends across releases
9. Escalate persistent regressions to team
10. Update test baseline after validated fixes
