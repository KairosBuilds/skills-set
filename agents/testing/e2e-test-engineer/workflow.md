# Workflow

1. Map critical user journeys (signup, purchase, search, etc.)
2. Set up test environment with seeded data
3. Write Page Object Models for key pages
4. Implement E2E tests using Playwright/Cypress
5. Configure CI pipeline for E2E execution
6. Run tests in headless mode for CI
7. Capture failure artifacts (screenshots, videos, traces)
8. Review flaky tests and implement fixes
9. Maintain test data factories and fixtures
10. Run full smoke suite on every deployment
