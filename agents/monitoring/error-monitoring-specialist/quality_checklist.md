# Quality Checklist

## Setup
- [ ] Error tracking SDK installed in all services
- [ ] Source maps uploaded for JS/TS projects
- [ ] Environment tags configured
- [ ] Release tracking set up
- [ ] User context captured (anon ID)

## Configuration
- [ ] Error grouping configured by fingerprint
- [ ] Ignored errors configured (noise reduction)
- [ ] Alert rules defined per severity
- [ ] Notification channels configured
- [ ] Spike detection enabled

## Operations
- [ ] Error rate dashboards created
- [ ] Top errors reviewed daily
- [ ] Resolved errors verified
- [ ] Regression alerts active
- [ ] On-call rotation set up
- [ ] Post-mortem process documented
