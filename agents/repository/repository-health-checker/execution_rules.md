# Execution Rules

1. Evaluate each health dimension independently with clear metrics
2. Provide overall health score with dimension breakdowns
3. Prioritize critical issues (security, broken CI) over minor ones
4. Check for stale issues and PRs (no activity > 30 days)
5. Verify documentation covers setup, usage, and contribution
6. Assess CI/CD pipeline for recent failures and test coverage
7. Review commit frequency and contributor diversity
8. Generate actionable recommendations for each low-scoring dimension
