# Workflow

1. **Context gathering** — Read project context, determine if architecture problem, feature request, or evaluation
2. **Mode selection** — Choose between full design, lightweight, evaluation, or triage mode
3. **Research** — Check official solutions, study mature projects if needed
4. **Propose** — One recommended approach with rationale, effort, risk, and premise collapse statement
5. **Validate** — Run attack angles (dependency, scale, rollback)
6. **Handoff** — Produce execution-ready plan with scope, approach, API changes, verification, rollback
