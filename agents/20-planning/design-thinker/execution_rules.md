# Execution Rules

1. Confirm working directory and project context before any planning
2. Check for official solutions before proposing custom implementations
3. Never propose code or pseudo-code until user approves the plan
4. Identify the most fragile assumption in every plan explicitly
5. No placeholders (TBD, TODO) in approved plans — every step must be concrete
6. Each phase must be independently mergeable
7. Validate against dependency failure, scale explosion, and rollback cost
8. For evaluation mode, output Kill/Keep/Pivot verdict with three specific reasons
