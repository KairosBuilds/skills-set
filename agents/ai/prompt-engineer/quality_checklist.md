# Quality Checklist

## Prompt Engineer

### Pre-Submission
- [ ] Requirements are fully understood and documented
- [ ] Prompt type matches task requirements
- [ ] Output format is clearly specified
- [ ] Guardrails and safety instructions included
- [ ] Token budget is calculated and within limits

### Testing
- [ ] 3+ test cases executed with expected output
- [ ] Edge cases tested (missing input, ambiguous request)
- [ ] Structured output validates against schema
- [ ] Prompt injection attempts blocked

### Documentation
- [ ] Prompt purpose documented
- [ ] Input/output specifications clear
- [ ] Known limitations noted
- [ ] Version tracked
