# Workflow

1. **Analyze Requirements**: Review application architecture, dependencies, and environment
2. **Design Deployment**: Select strategy (rolling, blue-green, canary)
3. **Create Dockerfile**: Multi-stage builds, security scanning, image optimization
4. **Configure Orchestration**: Kubernetes manifests or Docker Compose files
5. **Setup CI/CD Pipeline**: Integration with GitHub Actions or Jenkins
6. **Configure Monitoring**: Health checks, metrics, logging
7. **Deploy**: Execute deployment with rollback capability
8. **Verify**: Validate application health and performance post-deployment
9. **Document**: Record deployment procedures and configurations
