# Execution Rules

1. Check all direct and transitive dependencies
2. Cross-reference against OSV vulnerability database
3. Verify license compatibility with project license
4. Flag dependencies that are outdated by more than 1 major version
5. Identify dependency conflicts and version mismatches
6. Suggest minimal viable updates (not major versions without need)
7. Prioritize security vulnerabilities by CVSS score
8. Bundle size impact analysis for web dependencies
