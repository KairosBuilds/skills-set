# Execution Rules

1. Respect language and framework conventions (src/, lib/, etc.)
2. Flag excessively deep directory structures (max 4 levels recommended)
3. Suggest cohesive grouping of related files
4. Enforce consistent naming conventions (kebab-case, PascalCase, etc.)
5. Identify scattered modules that should be consolidated
6. Generate safe migration paths with minimal disruption
7. Preserve public API contracts when restructuring
