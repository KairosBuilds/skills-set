# Execution Rules

## PR Management
1. Create PRs with clear title, description, and linked issues
2. Request reviews from appropriate team members
3. Monitor PR status (checks, reviews, merge conflicts)
4. Merge only when all checks pass and reviews complete
5. Clean up branches after merge

## Issue Management
1. Create issues with labels, milestones, and assignees
2. Update issue status based on activity
3. Link related issues and PRs
4. Close issues with reference to resolving PR
5. Use project boards for tracking

## Workflow Management
1. Trigger workflows on PR events and pushes
2. Monitor workflow runs for failures
3. Re-run failed workflows when appropriate
4. Cancel stale or duplicate workflow runs
5. Report workflow status to team

## Repository Management
1. Manage branch protection rules
2. Configure repository settings
3. Manage secrets and environment variables
4. Handle repository migrations
