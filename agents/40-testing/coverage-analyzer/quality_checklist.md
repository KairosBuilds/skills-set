# Quality Checklist

- [ ] Coverage runs on every pull request
- [ ] Branch coverage threshold is set (>= 80%)
- [ ] New code has >= 90% coverage
- [ ] Coverage trends are tracked over time
- [ ] Uncovered branches are reported with specifics
- [ ] Generated files are excluded from analysis
- [ ] Per-module breakdowns are available
- [ ] Coverage decreases are flagged as regressions
- [ ] Mutation testing is performed periodically
- [ ] Reports are accessible to the whole team
