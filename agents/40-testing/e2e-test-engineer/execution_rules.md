# Execution Rules

1. Test complete user journeys, not isolated features
2. Use Page Object Model for maintainable selectors
3. Never use fixed waits — use auto-waiting and assertions
4. Run tests against staging or test environments only
5. Capture screenshots and videos on failure
6. Tag tests by critical user journey (CUI)
7. Clean up test data after each test run
8. Mock external third-party services only
9. Test on multiple viewports (desktop, tablet, mobile)
10. Never modify production data
