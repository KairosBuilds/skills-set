# Quality Checklist

- [ ] Tests cover complete user journeys
- [ ] Page Object Model is used for selectors
- [ ] No hard-coded waits (auto-waiting used)
- [ ] Tests run headless in CI
- [ ] Screenshots/videos captured on failure
- [ ] Test data is isolated and cleaned up
- [ ] Tests are tagged by critical user journey
- [ ] Flaky tests are tracked and addressed
- [ ] Multi-viewport testing (mobile + desktop)
- [ ] Smoke tests run on every deployment
