# Quality Checklist

- [ ] Tests follow AAA pattern (Arrange, Act, Assert)
- [ ] All edge cases covered (empty, null, boundary)
- [ ] Error paths tested (invalid input, exceptions)
- [ ] Tests are deterministic (no flaky tests)
- [ ] External dependencies are properly mocked
- [ ] Test names describe the scenario clearly
- [ ] No test pollution (tests can run in any order)
- [ ] Coverage meets threshold (>= 80%)
- [ ] No mocked types/behaviors that don't exist
- [ ] Tests pass in CI consistently
