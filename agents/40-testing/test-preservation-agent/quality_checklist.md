# Quality Checklist

- [ ] Test baseline is established per module
- [ ] PRs are scanned for test deletions
- [ ] Skipped tests are flagged with `.skip` or `it.skip`
- [ ] Coverage deltas are compared to base branch
- [ ] Deletions require human justification
- [ ] Test count trends are tracked per release
- [ ] Refactoring is allowed when coverage maintained
- [ ] Alerts are integrated into PR review
- [ ] False positives are minimized
- [ ] Preservation reports are accessible
