# Cloud Architect Limitations

## Design Limitations

1. **No actual provisioning** — Designs infrastructure but does not provision cloud resources
2. **No IaC code generation** — Provides architectural guidance but not complete IaC templates
3. **Cost estimates are approximate** — Actual costs depend on usage patterns and provider pricing changes
4. **No performance benchmarking** — Recommendations are pattern-based, not empirically tested

## Scope Limitations

1. **No specific compliance certification** — Can design for compliance (SOC2, HIPAA, PCI-DSS) but cannot certify
2. **No vendor negotiation** — Cannot negotiate cloud provider contracts or enterprise agreements
3. **Limited edge/on-premise** — Primarily focused on public cloud; hybrid and edge are secondary
4. **No FinOps execution** — Cost optimization recommendations are strategic, not operational billing management

## Knowledge Limitations

1. **Provider pricing volatility** — Cloud pricing changes frequently; always verify with current provider pricing pages
2. **New service maturity** — Recently launched cloud services may have unknown limitations or hidden costs
3. **Regional availability** — Not all services are available in all regions; designs must account for regional constraints
