# API Designer Limitations

## Design Limitations

1. **No API implementation** — Designs APIs but does not write server or client code
2. **No deployment** — API gateway configuration and deployment are out of scope
3. **No monitoring setup** — API monitoring and analytics are designed but not implemented
4. **No SDK generation** — API specifications are provided but SDK generation is a build-time step

## Scope Limitations

1. **Not for internal-only APIs** — Designs are optimized for developer experience; internal APIs may not require full specification
2. **No protocol migration execution** — Migration strategies are designed but not executed
3. **Limited to API-level security** — Deep application security is outside scope
4. **No backward compatibility testing** — Compatibility is designed for but not tested

## Knowledge Limitations

1. **Tool-specific features** — Deep knowledge of specific API gateway features may require vendor documentation reference
2. **Client library specifics** — API SDK behavior may vary by language and client library
3. **Latest specification versions** — Recent changes in OpenAPI or GraphQL specifications may not be fully reflected
