# Cloud Architect Workflow

## Standard Design Flow

### Phase 1: Requirements & Assessment
1. Review application requirements and constraints
2. Assess current infrastructure (for migration scenarios)
3. Identify compliance and regulatory requirements
4. Define success criteria

### Phase 2: Cloud Architecture Design
1. Select cloud provider(s) and services
2. Design network topology (VPC, subnets, connectivity)
3. Design compute architecture (instances, containers, serverless)
4. Design storage and data layer
5. Design security architecture (IAM, encryption, monitoring)
6. Design DR and backup strategy

### Phase 3: Migration Planning (if applicable)
1. Choose migration strategy per workload (6 Rs)
2. Plan migration waves and dependencies
3. Design cutover and rollback procedures
4. Plan testing and validation approach

### Phase 4: Cost Optimization
1. Estimate initial and ongoing costs
2. Identify optimization opportunities (RIs, spot, right-sizing)
3. Design auto-scaling policies
4. Create cost monitoring and alerting strategy

### Phase 5: Documentation
1. Create infrastructure architecture diagram
2. Document security and compliance architecture
3. Write migration roadmap
4. Create operations runbooks
5. Produce cost analysis report

### Phase 6: Review & Handoff
1. Present architecture to stakeholders
2. Address feedback and iterate
3. Handoff to DevOps for implementation

## Output Template

```
1. Executive Summary
2. Current State Assessment
3. Target Architecture
4. Network Design
5. Security Architecture
6. Compute & Storage Design
7. Data Architecture
8. Migration Roadmap
9. Cost Analysis
10. DR & Business Continuity
11. Operations Plan
12. ADRs
```
