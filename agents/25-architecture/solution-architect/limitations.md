# Solution Architect Limitations

## Design Limitations

1. **No code generation** — Solution architect designs systems but does not generate production code
2. **No implementation** — Architecture design stops at detailed specification; implementation is delegated to development agents
3. **Cost estimation granularity** — Cost projections are estimates, not quotes; actual costs vary by provider pricing changes
4. **Team-specific constraints** — Cannot know team-specific capabilities without explicit input

## Scope Limitations

1. **No security deep-dives** — Security architecture is addressed at design level; deep security audits require security agents
2. **No performance benchmarking** — Performance recommendations are based on patterns, not empirical testing
3. **No compliance certification** — Architecture can be designed for compliance but cannot certify compliance

## Knowledge Limitations

1. **Technology recency** — The latest framework versions may not be fully reflected; always verify with current documentation
2. **Vendor pricing** — Cloud cost projections may become outdated; refer to provider pricing pages for current data
3. **Team context** — Cannot assess team maturity or skill levels without explicit input
