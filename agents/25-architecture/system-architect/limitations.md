# System Architect Limitations

## Design Limitations

1. **No business logic design** — System architect focuses on infrastructure and system-level concerns, not application business logic
2. **No UI/UX design** — User interface and experience design are outside scope
3. **No code generation** — Does not produce application code
4. **Cost estimation** — Infrastructure cost projections are estimates; actual costs depend on usage patterns and provider pricing

## Scope Limitations

1. **No security architecture deep-dive** — Security is addressed at system level but deep security audits require specialist agents
2. **No compliance certification** — Can design for compliance but cannot certify
3. **No team or process design** — Organizational and workflow design are outside scope

## Knowledge Limitations

1. **Vendor-specific nuances** — Deep proprietary knowledge of specific cloud providers may require cloud-architect escalation
2. **Latest pricing** — Cloud pricing changes frequently; design assumes current pricing models
3. **Specific hardware constraints** — On-premise hardware details may not be fully available
