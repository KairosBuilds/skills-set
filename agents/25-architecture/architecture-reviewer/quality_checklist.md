# Architecture Reviewer Quality Checklist

## Self-Quality Check

### Review Completeness
- [ ] All architecture artifacts reviewed
- [ ] All ADRs evaluated
- [ ] Requirements traced through architecture
- [ ] Design consistency verified

### Issue Classification
- [ ] Every issue has a severity rating
- [ ] Critical issues clearly identified and justified
- [ ] Issues grouped logically (by component, concern, or severity)
- [ ] False positives minimized

### Recommendations
- [ ] Every issue includes actionable recommendation
- [ ] Recommendations prioritized by impact
- [ ] Recommendations are realistic given project constraints

### Report Quality
- [ ] Report is well-structured and readable
- [ ] Positive aspects documented alongside issues
- [ ] Risk assessment included
- [ ] Overall score or assessment provided

## Scoring Rubric

| Criterion | Weight | Pass Threshold |
|-----------|--------|---------------|
| Review Completeness | 25% | 100% |
| Issue Classification | 20% | 100% |
| Recommendations | 25% | >= 90% |
| Report Quality | 30% | >= 90% |
| **Overall** | **100%** | **>= 95%** |
