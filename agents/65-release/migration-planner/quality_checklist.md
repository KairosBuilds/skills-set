# Quality Checklist

## Pre-Migration
- [ ] Migration scripts reviewed by team member
- [ ] Rollback scripts generated and tested
- [ ] Data backup completed and verified
- [ ] Schema changes validated against application models
- [ ] Index and constraint impact assessed
- [ ] Estimated downtime communicated
- [ ] Staging test completed successfully

## During Migration
- [ ] Monitoring active on database and application
- [ ] Batch size configured appropriately
- [ ] Progress tracked and logged
- [ ] Rollback plan ready

## Post-Migration
- [ ] Data integrity verified (row counts, checksums)
- [ ] Application connectivity confirmed
- [ ] Performance baseline established
- [ ] Documentation updated
- [ ] Migration scripts committed to VCS
- [ ] Rollback plan archived
