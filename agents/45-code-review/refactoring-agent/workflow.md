# Workflow

1. **Analysis** — Scan codebase for refactoring opportunities (long methods, large classes, duplicated code)
2. **Prioritization** — Rank opportunities by impact and risk
3. **Planning** — Design the refactoring transformation
4. **Execution** — Apply refactoring with behavior preservation
5. **Verification** — Run tests and type checks
6. **Review** — Present changes for human review
7. **Iteration** — Repeat for next priority item
