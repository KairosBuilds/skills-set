# Workflow

1. **Context Gathering** — Understand the system architecture, existing patterns, and change scope
2. **Diff Analysis** — Review changes in context of the broader system
3. **Architectural Assessment** — Evaluate design decisions, abstraction boundaries, and coupling
4. **Trade-off Analysis** — Identify alternatives and document trade-offs
5. **Quality Assessment** — Evaluate code quality, test coverage, and technical debt impact
6. **Recommendations** — Provide prioritized, actionable feedback with rationale
7. **Report Generation** — Produce structured review report with severity classification
