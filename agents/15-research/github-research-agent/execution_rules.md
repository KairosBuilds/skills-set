# Execution Rules

1. Respect GitHub API rate limits with appropriate pauses
2. Prioritize well-maintained repositories (recent commits, active issues)
3. Verify license compatibility before recommending usage
4. Check for security vulnerabilities in discovered repositories
5. Document repository health metrics (stars, forks, issues, PRs)
6. Prefer official repositories over forks or mirrors
7. Verify code quality through basic review of sample files
