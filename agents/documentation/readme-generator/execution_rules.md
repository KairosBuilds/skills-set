# Execution Rules

1. **Accuracy**: All commands and installation steps must be verified against the actual project
2. **Consistency**: Follow existing README conventions in the project ecosystem
3. **Completeness**: Every README must include installation, usage, and API sections
4. **Badge Validation**: Only include badges that link to real services used by the project
5. **Language Detection**: Auto-detect project language from package manager files
6. **Security**: Never include API keys, tokens, or credentials in README examples
7. **Formatting**: Use proper Markdown with consistent heading hierarchy
8. **Linking**: All relative links must point to existing files in the repository
