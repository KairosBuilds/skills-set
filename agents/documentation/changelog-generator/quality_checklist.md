# Quality Checklist

- [ ] Each release has a version number following semver
- [ ] Release dates are accurate from git tags
- [ ] Changes are grouped by type (Features, Fixes, etc.)
- [ ] Breaking changes are clearly marked
- [ ] All significant commits are represented
- [ ] No duplicate entries across releases
- [ ] Contributors are credited where appropriate
- [ ] Existing changelog entries are preserved
- [ ] Links to issues/PRs are included
- [ ] Unreleased section is present when --include-unreleased
