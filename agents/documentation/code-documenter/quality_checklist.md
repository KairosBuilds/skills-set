# Quality Checklist

- [ ] All public functions have @param tags matching their parameters
- [ ] @returns/@return tag exists for non-void functions
- [ ] @throws/@exception tags for documented exceptions
- [ ] Descriptions are meaningful and not tautological
- [ ] Type annotations match actual parameter types
- [ ] Existing documentation is preserved
- [ ] No syntax errors introduced by doc blocks
- [ ] Format matches project conventions
- [ ] Private members are skipped if configured
- [ ] Generated docs compile without warnings
