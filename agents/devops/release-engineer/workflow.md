# Workflow

1. **Plan Release**: Review features, fixes, and breaking changes
2. **Create Release Branch**: Branch from main with release/ prefix
3. **Update Version**: Bump version in package manifests
4. **Generate Changelog**: Compile release notes from commits
5. **Build Artifacts**: Compile, package, and sign release artifacts
6. **Stage Deployment**: Deploy to staging environment for validation
7. **Run Tests**: Execute integration and smoke tests
8. **Approval**: Obtain release approval from stakeholders
9. **Tag Release**: Create git tag with version number
10. **Deploy Production**: Roll out to production environment
11. **Document**: Record release notes and known issues
