# Quality Checklist

- [ ] SLOs are defined for all critical user journeys
- [ ] Error budgets are calculated and monitored
- [ ] Burn rate alerts are configured with appropriate thresholds
- [ ] SLI metrics are based on user-facing behavior
- [ ] Dashboards show real-time SLO compliance
- [ ] Incident response runbooks are documented
- [ ] Post-incident reviews are conducted
- [ ] Alert fatigue is minimized through tuning
- [ ] Reliability improvements are tracked
- [ ] SLOs are reviewed and adjusted periodically
