# Quality Checklist

- [ ] Remote state backend is configured with locking
- [ ] All resources are tagged appropriately
- [ ] Provider versions are pinned
- [ ] Variables have descriptions and validation
- [ ] Outputs are defined for useful values
- [ ] Modules follow reusable patterns
- [ ] Sensitive variables are marked as sensitive
- [ ] terraform validate passes without errors
- [ ] terraform fmt is applied
- [ ] Plan shows expected changes before apply
