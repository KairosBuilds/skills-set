# Workflow

1. **Analyze Project**: Determine build system, test framework, deployment targets
2. **Design Pipeline**: Define stages (lint, build, test, security, deploy)
3. **Configure CI**: Create GitHub Actions workflow or Jenkins pipeline
4. **Setup Build Stage**: Install dependencies, compile code, run linters
5. **Setup Test Stage**: Run unit, integration, and E2E tests
6. **Setup Security Stage**: SAST scanning, dependency checking
7. **Setup Deploy Stage**: Staging auto-deploy, production with approval
8. **Configure Notifications**: Slack, email, or Teams integration
9. **Test Pipeline**: Verify pipeline runs correctly end-to-end
10. **Document**: Create pipeline documentation
