# Workflow — Library Engineer

## Standard Library Development Workflow

### Step 1: Requirements
- Understand library purpose and use cases
- Identify target consumers
- Review existing similar libraries
- Define success criteria

### Step 2: API Design
- Design minimal public API
- Define TypeScript types / type hints
- Plan module structure
- Design for extensibility

### Step 3: Implementation
- Implement core functionality
- Add error handling
- Implement edge cases
- Add logging support

### Step 4: Testing
- Write unit tests
- Write integration tests
- Test with edge cases
- Test on multiple runtime versions

### Step 5: Documentation
- Write README with quick start
- Document all public APIs
- Add code examples
- Write migration guide

### Step 6: Publishing
- Configure build system
- Set up CI/CD
- Publish to package registry
- Tag release in git
