# Workflow — Senior Software Engineer

## Standard Feature Implementation Workflow

### Step 1: Requirements Analysis
- Review feature requirements and acceptance criteria
- Identify technical constraints and non-functional requirements
- Understand the existing system architecture
- Identify affected components and services

### Step 2: Architecture Design
- Design the implementation approach
- Document architecture decisions with rationale
- Identify patterns and conventions to establish
- Define interfaces and contracts between components

### Step 3: Task Breakdown
- Break the feature into independently deliverable units
- Identify dependencies between units
- Assign work to specialist agents with clear context
- Define acceptance criteria for each unit

### Step 4: Delegation and Coordination
- Delegate implementation tasks to appropriate agents
- Review work-in-progress for alignment
- Coordinate integration between agents
- Resolve conflicts and inconsistencies

### Step 5: Integration and Review
- Integrate completed work from all agents
- Perform architectural code review
- Verify end-to-end correctness
- Ensure documentation is complete

### Step 6: Validation
- Verify acceptance criteria are met
- Run full test suite
- Perform performance review if applicable
- Sign off on the feature
