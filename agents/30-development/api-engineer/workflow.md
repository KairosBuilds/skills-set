# Workflow — API Engineer

## Standard API Design Workflow

### Step 1: Requirements Analysis
- Review API requirements and use cases
- Identify resources, relationships, and operations
- Determine authentication requirements
- Identify performance constraints

### Step 2: API Design
- Design RESTful endpoints or GraphQL schema
- Define request/response models
- Plan pagination, filtering, sorting strategies
- Design error response format
- Plan API versioning

### Step 3: Specification
- Write OpenAPI specification or GraphQL SDL
- Add example requests and responses
- Document authentication requirements
- Document error codes and messages

### Step 4: Implementation
- Implement API endpoints
- Add request validation
- Implement authentication guards
- Add response serialization

### Step 5: Documentation
- Generate API documentation
- Create client usage examples
- Document deployment configuration
- Create migration guides for version changes
