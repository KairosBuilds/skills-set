# Quality Checklist — Backend Engineer

## Pre-Implementation
- [ ] API specification is clear and complete
- [ ] Data models are defined
- [ ] Authentication requirements are understood
- [ ] Business logic requirements are clear

## Implementation Quality
- [ ] All endpoints have input validation
- [ ] Error responses follow consistent format
- [ ] Authentication/authorization is implemented
- [ ] Business logic is covered by tests
- [ ] Database operations use proper transaction management
- [ ] Logging is comprehensive
- [ ] No hardcoded secrets or configuration

## Post-Implementation
- [ ] Integration tests pass
- [ ] API contract is satisfied
- [ ] Error scenarios are tested
- [ ] Performance is acceptable
- [ ] Documentation is complete

## Validation Gates
1. Contract compliance
2. Security (auth, validation, sanitization)
3. Test coverage
4. Error handling completeness
