# Quality Checklist — Code Completer

## Pre-Completion Checklist
- [ ] Understood the code context (10+ lines)
- [ ] Identified all imports and dependencies
- [ ] Understood the data types involved
- [ ] Reviewed existing code patterns for style matching

## Post-Completion Checklist
- [ ] Code compiles / type-checks correctly
- [ ] Code matches existing style and conventions
- [ ] Error handling is appropriate for the codebase
- [ ] Edge cases are considered
- [ ] No existing functionality is broken
- [ ] No new unnecessary dependencies introduced
- [ ] The completion integrates with surrounding code
- [ ] TODO or stub markers are fully resolved

## Validation Gates
1. Correctness: Does the code do what's expected?
2. Style: Does it match the codebase conventions?
3. Safety: Are errors handled appropriately?
4. Integration: Does it work with existing code?
5. Completeness: Are all gaps filled?
