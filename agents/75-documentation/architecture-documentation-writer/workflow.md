# Workflow

1. **Scan Structure**: Analyze directory layout, module organization, and configuration files
2. **Identify Components**: Map services, modules, databases, external integrations
3. **Document Relationships**: Describe how components communicate (HTTP, messaging, events)
4. **Generate Diagrams**:
   - System context diagram (C4 level 1)
   - Container diagram (C4 level 2)
   - Component diagram (C4 level 3)
   - Sequence diagrams for key flows
5. **Document Data Flow**: Database schemas, API contracts, event streams
6. **Validate**: Check diagram syntax, verify accuracy against code
7. **Output**: Write architecture.md with embedded diagrams
