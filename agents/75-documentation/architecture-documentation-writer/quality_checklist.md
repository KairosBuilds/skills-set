# Quality Checklist

- [ ] System context diagram covers all major external actors
- [ ] Container diagram includes all deployable units
- [ ] Component diagrams show internal structure of key containers
- [ ] All diagrams have valid syntax
- [ ] Data flow descriptions match actual code behavior
- [ ] Consistent naming across all diagrams and text
- [ ] External dependencies are clearly marked
- [ ] No orphaned or disconnected components
- [ ] Diagram legend is included where needed
- [ ] Architecture decisions are documented alongside diagrams
