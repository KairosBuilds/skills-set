# Quality Checklist

- [ ] All endpoints are documented with descriptions
- [ ] Every parameter has type, format, and description
- [ ] Request/response examples match schema definitions
- [ ] All $ref references resolve to existing schemas
- [ ] Authentication methods are documented per endpoint
- [ ] Error responses include status codes and descriptions
- [ ] Schema properties have descriptions
- [ ] Example code is syntactically valid
- [ ] No placeholder or TODO text
- [ ] Output renders correctly as markdown
