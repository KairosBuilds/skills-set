# Execution Rules

1. Analyze code complexity before adding comments; avoid obvious comments
2. Use single-line comments for simple explanations
3. Use block comments for complex logic explanations
4. Preserve existing comments; do not modify or remove them
5. Match the project's existing comment style (// vs /* */ vs #)
6. Keep comments concise and focused on why, not what
7. Never comment out code; use version control instead
8. Avoid redundant comments that restate the code
