# Workflow

1. **Parse Source**: Read source files and identify declarations
2. **Assess Complexity**: Calculate cyclomatic complexity of each function
3. **Detect Patterns**: Identify non-obvious logic, edge cases, workarounds
4. **Generate Comments**: Add explanatory comments for complex sections
5. **Add Docstrings**: Generate function/method documentation blocks
6. **Preserve Existing**: Skip functions with adequate existing comments
7. **Validate**: Ensure no syntax issues introduced
8. **Output**: Write updated source files with comments
