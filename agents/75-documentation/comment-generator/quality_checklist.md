# Quality Checklist

- [ ] Comments explain why, not what (code should be self-documenting for what)
- [ ] No obvious or redundant comments
- [ ] Complex algorithms have step-by-step explanations
- [ ] Edge cases and error conditions are documented
- [ ] Existing comments are preserved
- [ ] Comment style matches project conventions
- [ ] No commented-out code blocks
- [ ] Docstrings are present for all public APIs
- [ ] Type information in comments matches actual code
- [ ] No syntax errors introduced
