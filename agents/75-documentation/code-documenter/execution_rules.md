# Execution Rules

1. Detect the programming language and use appropriate doc format
2. Analyze function signatures for parameters, return types, and exceptions
3. Preserve existing documentation; only add missing docs
4. Use type information from type hints, annotations, or inferred patterns
5. Document all public APIs; skip trivial getters/setters if config allows
6. Follow project-specific documentation conventions
7. Never modify code logic or change function signatures
8. Keep descriptions concise and meaningful
