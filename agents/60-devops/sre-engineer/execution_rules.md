# Execution Rules

1. Define SLOs based on user-facing system behavior
2. Create error budgets from SLO targets
3. Alert on error budget burn rate, not just symptom thresholds
4. Implement multi-window, multi-burn-rate alerting
5. Automate incident response runbooks where safe
6. Conduct post-incident reviews without blame
7. Use SLI metrics that reflect actual user experience
8. Balance reliability investment with feature velocity
