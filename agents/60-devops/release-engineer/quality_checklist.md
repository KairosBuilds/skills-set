# Quality Checklist

- [ ] Release version follows semver
- [ ] Release branch is created from main
- [ ] Changelog is updated with all changes
- [ ] Release artifacts are signed and stored
- [ ] Staging deployment is validated
- [ ] Release approval is obtained
- [ ] Git tag matches release version
- [ ] Release notes document known issues
- [ ] Rollback plan is documented
- [ ] Dependencies are pinned and verified
