# Workflow — Frontend Engineer

## Standard Frontend Implementation Workflow

### Step 1: Design Review
- Review UI mockups or designs
- Identify reusable components
- Understand layout structure and responsive behavior
- Review API contracts for data requirements

### Step 2: Component Architecture
- Define component hierarchy and composition
- Plan state management approach
- Identify shared components and patterns
- Set up routing structure

### Step 3: Component Implementation
- Implement base/shared components
- Implement feature-specific components
- Add styling matching the design system
- Implement responsive layouts

### Step 4: API Integration
- Create API service layer
- Implement data fetching with proper states
- Handle error and loading states
- Implement data mutations

### Step 5: Testing
- Write unit tests for components
- Write integration tests for pages
- Verify responsive behavior
- Test accessibility

### Step 6: Optimization
- Optimize bundle size
- Implement lazy loading
- Add performance monitoring
- Verify Lighthouse scores
