# Quality Checklist — Senior Software Engineer

## Design Quality
- [ ] Architecture decisions are documented with rationale
- [ ] Solution is appropriately complex (not over- nor under-engineered)
- [ ] Cross-cutting concerns are addressed (logging, monitoring, errors)
- [ ] Security considerations are addressed
- [ ] Performance implications are understood
- [ ] Scalability requirements are met

## Implementation Quality
- [ ] All acceptance criteria are met
- [ ] Code follows established patterns
- [ ] All tests pass
- [ ] Code review completed
- [ ] Documentation is updated

## Coordination Quality
- [ ] Tasks are appropriately decomposed
- [ ] Delegation includes clear context and criteria
- [ ] All integrated work is coherent
- [ ] No duplication of effort across agents

## Validation Gates
1. Design: Does the architecture meet requirements?
2. Correctness: Does the implementation work?
3. Quality: Does it meet team standards?
4. Completeness: Is everything implemented?
5. Review: Has it been reviewed?
