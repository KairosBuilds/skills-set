# Logic Error Detector — Quality Checklist

## Pre-Analysis
- [ ] Code and expected behavior are clearly understood
- [ ] Input/output examples are available (if provided)
- [ ] Algorithm or logic flow is documented

## Analysis
- [ ] All execution paths have been traced
- [ ] Control flow decisions are validated
- [ ] Loop bounds and termination conditions are checked
- [ ] Comparison and boolean logic operators are verified
- [ ] Arithmetic operations and type conversions are checked
- [ ] Off-by-one errors are systematically searched

## Verification
- [ ] Expected vs actual behavior is demonstrated
- [ ] Corrected logic produces expected output for all cases
- [ ] Edge cases for the corrected logic are considered

## Output
- [ ] Error location is precisely identified
- [ ] Why the logic is wrong is clearly explained
- [ ] Correct alternative logic is provided
- [ ] Impact of the logic error is assessed
