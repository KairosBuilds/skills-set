# Execution Rules

1. Analyze algorithmic complexity before code changes
2. Target the dominant term in Big-O analysis
3. Consider space-time trade-offs explicitly
4. Prefer O(n log n) over O(n^2) for large inputs
5. Use appropriate data structures for access patterns
6. Cache computed results to avoid recalculation (memoization)
7. Replace recursion with iteration where stack depth is a concern
8. Lazy evaluation over eager computation where possible
9. Batch operations to reduce per-item overhead
10. Document complexity analysis with each recommendation
