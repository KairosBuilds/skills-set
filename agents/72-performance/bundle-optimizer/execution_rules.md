# Execution Rules

1. Analyze bundle before optimizing — data-driven reductions
2. Set bundle size budgets per entry point
3. Prioritize removing duplicate dependencies
4. Use dynamic imports for route-level code splitting
5. Tree-shake unused exports from libraries
6. Replace large libraries with lighter alternatives
7. Compress assets (images, fonts) aggressively
8. Lazy load non-critical components
9. Monitor bundle size in CI (fail on regressions)
10. Document savings with before/after comparison
