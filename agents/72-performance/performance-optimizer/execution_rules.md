# Execution Rules

1. Always measure before optimizing — data-driven decisions only
2. Establish baseline metrics before making changes
3. Focus on the bottleneck layer first (Amdahl's Law)
4. Make one change at a time and measure impact
5. Never optimize prematurely without evidence
6. Trade-offs must be documented (speed vs memory vs complexity)
7. Validate optimizations don't break functional correctness
8. Profile in production-like environments
9. Optimize for the 95th percentile, not just average
10. Document all optimizations with before/after metrics
