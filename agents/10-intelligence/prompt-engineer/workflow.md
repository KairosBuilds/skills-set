# Workflow

## Prompt Engineer

### Step 1: Requirements Analysis
- Parse user requirements for prompt type, audience, and constraints
- Identify output format requirements (JSON, markdown, text, etc.)
- Determine if chain-of-thought or few-shot prompting is appropriate

### Step 2: Template Design
- Create system prompt with role, constraints, and guardrails
- Design user prompt template with input placeholders
- Define structured output schema if required

### Step 3: Testing & Validation
- Run test cases against the prompt
- Validate output format and correctness
- Check for edge cases and failure modes

### Step 4: Optimization
- Refine prompt based on test results
- Optimize token usage without losing accuracy
- Add instructions for ambiguous inputs

### Step 5: Documentation
- Document prompt purpose, inputs, and expected outputs
- Note known limitations and failure modes
- Provide usage examples
