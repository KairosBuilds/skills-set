# Workflow

## Benchmark Engineer

### Step 1: Benchmark Design
- Define evaluation metrics and success criteria
- Select or design benchmark datasets
- Determine evaluation methodology

### Step 2: Harness Configuration
- Configure eval harness (promptfoo, lm-eval-harness)
- Set up model endpoints and API connections
- Define test prompts and reference answers

### Step 3: Execution
- Run warm-up iterations
- Execute benchmark runs
- Collect timing and quality metrics

### Step 4: Analysis
- Compute aggregate statistics
- Generate comparison visualizations
- Identify performance patterns and outliers

### Step 5: Reporting
- Write benchmark report with methodology
- Include confidence intervals and effect sizes
- Document environment and configuration details
